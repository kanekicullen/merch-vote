# Голосование за принты — как запустить

Всё бесплатно, займёт ~10 минут. Никакого кода писать не надо.

## Что получишь

- `index.html` — страница для друзей. Показывает две картинки, они выбирают лучшую. Промежуточных результатов **не видно**.
- `admin.html` — только для тебя. Загружаешь эскизы, смотришь текущий топ, удаляешь неудачные.

---

## Шаг 1. Supabase (бэкенд для голосов)

1. Зайди на https://supabase.com → **Sign up** (можно через GitHub / Google).
2. **New project** → назови `merch-vote` (или как хочешь), сгенерируй пароль для БД, выбери регион поближе (Frankfurt / Stockholm). Жди ~2 минуты пока прогрузится.
3. Открой в левом меню **SQL Editor** → **New query**. Скопируй **весь** `schema.sql` из этой папки, вставь и нажми **Run**.
4. Придумай себе секрет для админки (просто длинную строку, например 20+ случайных символов). Замени в SQL строку `'ЗАМЕНИ_МЕНЯ_НА_ДЛИННУЮ_СЛУЧАЙНУЮ_СТРОКУ'` **до** того как нажмёшь Run. Если уже нажал — открой в SQL Editor и запусти:
   ```sql
   update app_config set value = 'ТВОЙ_НАСТОЯЩИЙ_СЕКРЕТ' where key = 'admin_secret';
   ```
5. В левом меню открой **Project Settings → Data API** (или **API**). Скопируй оттуда:
   - **Project URL** (например `https://abcdefg.supabase.co`)
   - **anon / public** key (длинный `eyJ...`)

## Шаг 2. Впиши ключи в `config.js`

Открой `config.js` любым редактором и подставь свои значения:
```js
window.CONFIG = {
  SUPABASE_URL:      'https://abcdefg.supabase.co',
  SUPABASE_ANON_KEY: 'eyJ...длинный.ключ...',
};
```

## Шаг 3. Опубликуй сайт (Netlify Drop)

1. Зайди на https://app.netlify.com/drop
2. Просто перетащи туда **папку `merch-vote`** целиком.
3. Через 10 секунд получишь ссылку вида `https://random-name-123.netlify.app/`.
4. (По желанию) на странице сайта в панели Netlify — **Site settings → Change site name** — можно поменять на что-то читаемое, например `nashi-printy.netlify.app`.

## Шаг 4. Загрузи эскизы

1. Открой в браузере `https://твой-сайт.netlify.app/admin.html`
2. Введи свой admin secret (тот, что был в SQL).
3. Перетащи файлы эскизов в область загрузки. Можно много сразу. Имя файла станет названием эскиза (переименуй файлы заранее, если нужны красивые названия).

## Шаг 5. Раскидай ссылку друзьям

Отдай друзьям **только** `https://твой-сайт.netlify.app/` (без `admin.html`).
Они не увидят ни рейтинг, ни счётчик голосов — только вопрос «какой лучше?»

Себе сохрани `https://твой-сайт.netlify.app/admin.html` — открывать всегда с секретом.

## Как это работает

- **Формат:** парные сравнения. Каждый раз показывается случайная пара эскизов. Друг тыкает лучший.
- **Рейтинг:** для каждого эскиза считается «сглаженная доля побед» — Байесов приор из 5 виртуальных побед и 5 поражений. Это чтобы дизайн с результатом «1 победа из 1 матча» не оказывался на первом месте с 100%.
- **Один голосующий = одна сессия:** в браузере пишется session_id. Пары, которые уже показали одному голосующему, повторно ему не покажутся (пока не переберёт все).
- **Клавиатура:** ← левый, → правый, `S` — пропустить пару.
- **Обновление админки:** результаты сами перезагружаются раз в 15 секунд. Или кнопка «Обновить».

## Важно про безопасность (для «дружеского» уровня)

- Секрет админа хранится **только в Supabase**, никогда не приходит на страницу голосования.
- Функция получения топа (`get_leaderboard`) требует секрет — без него никто не узнает промежуточные результаты, даже с devtools.
- **НО:** страница `admin.html` лежит на том же домене. Если друг введёт `/admin.html` вручную — увидит форму, но без секрета не пройдёт. Всё ок.
- **НО2:** эта конфигурация разрешает любому с ключом anon добавлять новые эскизы и голоса. Для дружеского опроса это норма. Если внезапно кто-то начнёт спамить — открой Supabase → Table Editor → почисти.

## Если хочется закончить голосование

В Supabase → Table Editor → таблица `designs`. Правишь как удобно. Голоса живут в `votes`.
Топ можешь смотреть в админке хоть по завершении, хоть в процессе.

## Возможные проблемы

- **«Что-то пошло не так» на главной странице** — проверь `config.js`: правильные URL и anon key. Также проверь что SQL из `schema.sql` действительно прошёл (в Supabase → SQL Editor должно быть «Success. No rows returned»).
- **Картинки не грузятся** — открой картинку по URL из админки в новой вкладке. Если ошибка `Object not found` — не создался бакет `sketches`. Перезапусти секцию про storage из `schema.sql`.
- **«Неверный секрет»** — сверься с тем, что реально лежит в `app_config`. В Supabase → SQL Editor: `select value from app_config where key='admin_secret';`

## Стоимость

Всё в бесплатных лимитах:
- Supabase бесплатно: 500 MB БД + 1 GB storage + 50 000 месячных активных юзеров. Для голосования друзей хватит на годы.
- Netlify бесплатно: 100 GB трафика в месяц.
